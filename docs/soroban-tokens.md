<header><h1>Stellar Assets and Contract Tokens</h1></header>
<p>Tokens exist in two forms on Stellar:</p>
<ol>
<li class="">Assets issued by Stellar accounts (<code>G...</code> addresses) and their built-in <a class="" href="/docs/tokens/stellar-asset-contract">Stellar Asset Contract (SAC)</a> implementation, and</li>
<li class=""><a class="" href="/docs/tokens/token-interface">Contract tokens</a> issued by a deployed WASM contract (<code>C...</code> addresses).</li>
</ol>
<p>Several factors can help you determine whether to issue an asset on Stellar or create a contract token with a smart contract for your project.</p>
<p>However:</p>
<h3 class="anchor anchorTargetStickyNavbar_Vzrq" id="tldr">TL;DR<a href="#tldr" class="hash-link" aria-label="Direct link to TL;DR" title="Direct link to TL;DR" translate="no">​</a></h3>
<p>If possible, we recommend issuing a Stellar asset and using the SAC to interact with that asset in smart contracts or to send to contract addresses. More on why below.</p>
<h2 class="anchor anchorTargetStickyNavbar_Vzrq" id="issuing-assets-on-stellar">Issuing assets on Stellar<a href="#issuing-assets-on-stellar" class="hash-link" aria-label="Direct link to Issuing assets on Stellar" title="Direct link to Issuing assets on Stellar" translate="no">​</a></h2>
<p>Stellar has first-class support for asset tokenization — issuing an asset can be done using a <a class="" href="/docs/tokens/quickstart">built-in transaction</a> without the development of a smart contract.</p>
<p>Stellar’s transactions are fast and cost-effective, making the network great for remittances and micropayments. It also has built-in features for compliance, asset management, and auditing. If you are looking to perform transfers of value, issuing assets on Stellar has all the needed capabilities.</p>
<p>Stellar assets:</p>
<ul>
<li class="">Are compatible with Stellar ecosystem products (such as Stellar wallets) and other ecosystem products (such as exchanges).</li>
<li class="">Benefit from <a class="" href="/docs/learn/fundamentals/anchors">anchors</a>, the bridges between the Stellar network and traditional financial systems. Explore the global <a href="https://anchors.stellar.org/" target="_blank" rel="noopener noreferrer" class="">Stellar anchor directory</a> for further details.</li>
<li class="">Give the issuer granular control over asset management with features that allow the issuer to <a class="" href="/docs/tokens/control-asset-access#naming-an-asset">name the asset</a>, <a class="" href="/docs/tokens/control-asset-access#controlling-access-to-an-asset-with-flags">determine access control</a>, <a class="" href="/docs/tokens/control-asset-access#limiting-the-supply-of-an-asset">limit asset supply</a>, <a class="" href="/docs/tokens/publishing-asset-info">publish asset information</a>, and <a class="" href="/docs/tokens/anatomy-of-an-asset#compliance">ensure compliance</a>.</li>
</ul>
<div class="theme-admonition theme-admonition-note admonition_xJq3 alert alert--secondary"><div class="admonitionHeading_Gvgb"><span class="admonitionIcon_Rf37"><svg viewBox="0 0 14 16"><path fill-rule="evenodd" d="M6.3 5.69a.942.942 0 0 1-.28-.7c0-.28.09-.52.28-.7.19-.18.42-.28.7-.28.28 0 .52.09.7.28.18.19.28.42.28.7 0 .28-.09.52-.28.7a1 1 0 0 1-.7.3c-.28 0-.52-.11-.7-.3zM8 7.99c-.02-.25-.11-.48-.31-.69-.2-.19-.42-.3-.69-.31H6c-.27.02-.48.13-.69.31-.2.2-.3.44-.31.69h1v3c.02.27.11.5.31.69.2.2.42.31.69.31h1c.27 0 .48-.11.69-.31.2-.19.3-.42.31-.69H8V7.98v.01zM7 2.3c-3.14 0-5.7 2.54-5.7 5.68 0 3.14 2.56 5.7 5.7 5.7s5.7-2.55 5.7-5.7c0-3.15-2.56-5.69-5.7-5.69v.01zM7 .98c3.86 0 7 3.14 7 7s-3.14 7-7 7-7-3.12-7-7 3.14-7 7-7z"></path></svg></span>note</div><div class="admonitionContent_BuS1"><p>Note that while these items are also possible with smart contract tokens, it is more work to build the token contract rather than using the already-implemented features of Stellar asset tokens.</p></div></div>
<p>Assets issued on the Stellar network are accessible to smart contracts with the use of that asset’s Stellar Asset Contract (SAC).</p>
<h3 class="anchor anchorTargetStickyNavbar_Vzrq" id="stellar-asset-contract">Stellar Asset Contract<a href="#stellar-asset-contract" class="hash-link" aria-label="Direct link to Stellar Asset Contract" title="Direct link to Stellar Asset Contract" translate="no">​</a></h3>
<p>The Stellar Asset Contract (SAC) is compiled into the protocol layer and allows smart contracts to interact with assets issued on Stellar. An instance of the SAC can be deployed for every Stellar asset by anyone who wants to interact with the asset from a contract. The SAC has access to all account balances (for XLM) and trustline balances (for all other assets) as well as smart contract token balances.</p>
<p>Read more about the SAC <a class="" href="/docs/tokens/stellar-asset-contract">here</a>.</p>
<p>Learn how to deploy a Stellar Asset Contract for an asset in <a class="" href="/docs/tools/cli/cookbook/deploy-stellar-asset-contract">this How-To Guide</a>.</p>
<p><strong>Benefits of the SAC:</strong></p>
<ul>
<li class="">Compatibility: the SAC benefits from Stellar assets&#x27; existing interoperability.</li>
<li class="">Cost and resource efficiency: the SAC is built into the protocol instead of being a contract that runs in a virtual machine. Each function within the SAC will be more resource-efficient than its contract-coded counterpart.</li>
<li class="">Less work: you don’t have to write an entirely new contract. A Stellar asset’s SAC already exists on the network and just needs to be deployed to be used.</li>
<li class="">Customization: Admin addresses can be contracts. Asset issuers can set a different smart contract as an admin for their asset’s SAC. Making the admin another smart contract allows the addition of custom and decentralized logic for the assets admin capabilities, such as authorizing balances and trust lines, minting tokens, etc.</li>
</ul>
<p><strong>Downside of the SAC:</strong></p>
<ul>
<li class="">Other than the customization noted above, it is not possible to modify the behavior of Stellar assets or their SAC. If you’re looking to use assets in a way not supported by Stellar assets, you can create your own smart contract token using the token interface and all applications that interact with tokens using the token interface will be able to interact with the contract token.</li>
</ul>
<h2 class="anchor anchorTargetStickyNavbar_Vzrq" id="contract-tokens">Contract tokens<a href="#contract-tokens" class="hash-link" aria-label="Direct link to Contract tokens" title="Direct link to Contract tokens" translate="no">​</a></h2>
<p>If you have a unique use case where the capabilities Stellar Assets are not sufficient, you can create a contract token that implements the <a class="" href="/docs/tokens/token-interface">token interface</a>. The token interface specifies the functions and events a contract must implement to be compatible with applications that use tokens.</p>
<p>The SAC also implements the token interface and applications that interoperate with the token interface can seamlessly interact with Stellar assets and contract tokens.</p>
<div class="theme-admonition theme-admonition-note admonition_xJq3 alert alert--secondary"><div class="admonitionHeading_Gvgb"><span class="admonitionIcon_Rf37"><svg viewBox="0 0 14 16"><path fill-rule="evenodd" d="M6.3 5.69a.942.942 0 0 1-.28-.7c0-.28.09-.52.28-.7.19-.18.42-.28.7-.28.28 0 .52.09.7.28.18.19.28.42.28.7 0 .28-.09.52-.28.7a1 1 0 0 1-.7.3c-.28 0-.52-.11-.7-.3zM8 7.99c-.02-.25-.11-.48-.31-.69-.2-.19-.42-.3-.69-.31H6c-.27.02-.48.13-.69.31-.2.2-.3.44-.31.69h1v3c.02.27.11.5.31.69.2.2.42.31.69.31h1c.27 0 .48-.11.69-.31.2-.19.3-.42.31-.69H8V7.98v.01zM7 2.3c-3.14 0-5.7 2.54-5.7 5.68 0 3.14 2.56 5.7 5.7 5.7s5.7-2.55 5.7-5.7c0-3.15-2.56-5.69-5.7-5.69v.01zM7 .98c3.86 0 7 3.14 7 7s-3.14 7-7 7-7-3.12-7-7 3.14-7 7-7z"></path></svg></span>note</div><div class="admonitionContent_BuS1"><p>Smart contracts cannot use Stellar assets unless that Stellar asset has a deployed SAC. Anyone can deploy the SAC for a Stellar asset to its reserved address.</p></div></div>
<p><strong>These example scenarios are not possible with the SAC and demonstrate what you could use the token interface for:</strong></p>
<ul>
<li class="">As the creator of a new token, you decide to implement a feature within your token smart contract that enables you to receive a 1% fee from every transaction involving your token. Whenever someone transfers your token to another user, 1% of the transferred amount is automatically deducted and sent to a designated wallet address that you control.</li>
<li class="">You want to develop a factory contract that automates the creation of instances of a specific token. This contract serves as a centralized and standardized way to deploy new token contracts on demand without manual intervention each time a new instance is needed.</li>
</ul>
