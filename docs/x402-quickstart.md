<p>When a client requests your endpoint, your server responds with a <code>402 Payment Required</code> status. The response includes headers that describe the payment requirements so that a compliant client can build a signed payment payload and retry the request.</p>
<p>Behind the scenes, the facilitator handles verification and onchain settlement. You do not need to run your own blockchain nodes or build raw transactions yourself. A managed facilitator service takes the signed payment payload from the client, verifies it against the specified network and asset, settles the transaction, and returns verification to your server so it can complete the request.</p>
<p>The client generates a signed payment payload that authorizes a stablecoin transfer, the facilitator verifies and settles that payload on the Stellar network, and the server returns a <code>200</code> response with the protected data.</p>
<p>The funds settle directly to the wallet address in <code>PAY_TO</code>.</p>

<h3 class="anchor anchorTargetStickyNavbar_Vzrq" id="setting-up-a-testnet-wallet">Setting up a testnet wallet<a href="#setting-up-a-testnet-wallet" class="hash-link" aria-label="Direct link to Setting up a testnet wallet" title="Direct link to Setting up a testnet wallet" translate="no">​</a></h3>
<p>Next, create a fresh account and fund it with testnet XLM and testnet USDC using Stellar Lab:</p>
<ul>
<li class=""><a href="https://lab.stellar.org/account/fund" target="_blank" rel="noopener noreferrer" class="">Fund a testnet account in Stellar Lab</a></li>
</ul>
<ol>
<li class="">Create a new keypair using Stellar Lab: <a href="https://lab.stellar.org/account/create" target="_blank" rel="noopener noreferrer" class="">https://lab.stellar.org/account/create</a></li>
<li class="">Fund with testnet XLM (Native Stellar Token): <a href="https://lab.stellar.org/account/fund" target="_blank" rel="noopener noreferrer" class="">https://lab.stellar.org/account/fund</a></li>
<li class="">Create the USDC trustline (there&#x27;s a button on the fund page above), sign and submit that transaction</li>
<li class="">Visit the circle faucet, select Stellar Testnet from the networks and add your public key for the wallet address input: <a href="https://faucet.circle.com" target="_blank" rel="noopener noreferrer" class="">https://faucet.circle.com</a></li>
</ol>
<p>Create a local environment file for the client and add your Stellar testnet secret key:</p>
<div class="language-bash codeBlockContainer_Ckt0 theme-code-block" style="--prism-color:#393A34;--prism-background-color:#f6f8fa"><div class="codeBlockTitle_OeMC">.env</div><div class="codeBlockContent_QJqH"><pre tabindex="0" class="prism-code language-bash codeBlock_bY9V thin-scrollbar" style="color:#393A34;background-color:#f6f8fa"><code class="codeBlockLines_e6Vv"><span class="token-line" style="color:#393A34"><span class="token assign-left variable" style="color:#36acaa">STELLAR_PRIVATE_KEY</span><span class="token operator" style="color:#393A34">=</span><span class="token plain">S</span><span class="token punctuation" style="color:#393A34">..</span><span class="token plain">.</span><br></span></code></pre></div></div>
<p>Edit <code>.env</code> and replace <code>S...</code> with the secret key for the account that will sign transactions and pay for the resource.</p>
<div class="theme-admonition theme-admonition-caution admonition_xJq3 alert alert--warning"><div class="admonitionHeading_Gvgb"><span class="admonitionIcon_Rf37"><svg viewBox="0 0 16 16"><path fill-rule="evenodd" d="M8.893 1.5c-.183-.31-.52-.5-.887-.5s-.703.19-.886.5L.138 13.499a.98.98 0 0 0 0 1.001c.193.31.53.501.886.501h13.964c.367 0 .704-.19.877-.5a1.03 1.03 0 0 0 .01-1.002L8.893 1.5zm.133 11.497H6.987v-2.003h2.039v2.003zm0-3.004H6.987V5.987h2.039v4.006z"></path></svg></span>caution</div><div class="admonitionContent_BuS1"><p>Secret keys provide full access to any digital assets held within the wallet. Use <code>.env</code> files only for hot wallets in testnet deployments.</p></div></div>
<h2 class="anchor anchorTargetStickyNavbar_Vzrq" id="run-the-client">Run The Client<a href="#run-the-client" class="hash-link" aria-label="Direct link to Run The Client" title="Direct link to Run The Client" translate="no">​</a></h2>
<p>Once the account is funded and the secret key is in <code>.env</code>, run the client in a second terminal:</p>
<div class="language-bash codeBlockContainer_Ckt0 theme-code-block" style="--prism-color:#393A34;--prism-background-color:#f6f8fa"><div class="codeBlockContent_QJqH"><pre tabindex="0" class="prism-code language-bash codeBlock_bY9V thin-scrollbar" style="color:#393A34;background-color:#f6f8fa"><code class="codeBlockLines_e6Vv"><span class="token-line" style="color:#393A34"><span class="token function" style="color:#d73a49">node</span><span class="token plain"> client.js</span><br></span></code></pre></div></div>
<p>This sends <code>$0.01</code> of testnet USDC from the client account to the server account.</p>
<p>The client then receives the protected JSON data: <code>{ secret: &quot;valuable content&quot; }</code></p>
<p>The server is set up as an MVP for machine-to-machine payments. If you want to enable human payments, the next step is to build a paywall. Take a look at the demo code for examples: <a href="https://github.com/stellar/x402-stellar/tree/main/examples/simple-paywall" target="_blank" rel="noopener noreferrer" class="">https://github.com/stellar/x402-stellar/tree/main/examples/simple-paywall</a></p>
