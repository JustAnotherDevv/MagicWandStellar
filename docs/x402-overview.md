class="anchor anchorTargetStickyNavbar_Vzrq" id="what-is-x402">What is x402?<a href="#what-is-x402" class="hash-link" aria-label="Direct link to What is x402?" title="Direct link to What is x402?" translate="no">​</a></h2>
<p>x402 is an open protocol from the Coinbase Developer Platform that enables programmatic, per request payments over HTTP, designed especially for AI agents and APIs. It effectively turns the old “402 Payment Required” HTTP status code into something usable, for both humans and AI agents.</p>
<p>On Stellar, x402 works with Soroban authorization so that clients can pay for API requests via signed auth entries, ideal for micropayments and payment enabled apps. To build an x402-enabled service or integrate payments into your app, see <a class="" href="/docs/build/apps">Build Applications</a> and the resources below.</p>

<p>Try the <a href="https://stellar.org/x402-demo" target="_blank" rel="noopener noreferrer" class="">x402 Demo</a>. Send an x402 payment on Stellar Testnet or Mainnet and see the x402 payment flow in action. (<a href="https://github.com/stellar/x402-stellar" target="_blank" rel="noopener noreferrer" class="">source code</a>)</p>

<h2 class="anchor anchorTargetStickyNavbar_Vzrq" id="x402-compatible-wallets">x402 Compatible Wallets<a href="#x402-compatible-wallets" class="hash-link" aria-label="Direct link to x402 Compatible Wallets" title="Direct link to x402 Compatible Wallets" translate="no">​</a></h2>
<p>To support x402 on Stellar, a wallet must support <a class="" href="/docs/build/guides/transactions/signing-soroban-invocations#method-2-auth-entry-signing">auth-entry signing</a> (Soroban authorization entry signing). The following wallets support auth-entry signing:</p>

<li class="">Albedo</li>
<li class="">Hana</li>
<li class="">HOT</li>
<li class="">Klever</li>
<li class="">OneKey</li>
</ul>

<h2 class="anchor anchorTargetStickyNavbar_Vzrq" id="x402-facilitators">x402 Facilitators<a href="#x402-facilitators" class="hash-link" aria-label="Direct link to x402 Facilitators" title="Direct link to x402 Facilitators" translate="no">​</a></h2>
<p>You can use a facilitator to verify and settle x402 payments. Two options are available for Stellar:</p>
<h3 class="anchor anchorTargetStickyNavbar_Vzrq" id="coinbase-x402-facilitator">Coinbase x402 Facilitator<a href="#coinbase-x402-facilitator" class="hash-link" aria-label="Direct link to Coinbase x402 Facilitator" title="Direct link to Coinbase x402 Facilitator" translate="no">​</a></h3>
<p>Coinbase’s x402 facilitator supports Stellar on <strong>Testnet</strong> with sponsored fees. Check which networks and options are supported:</p>
<ul>
<li class=""><strong><a href="https://www.x402.org/facilitator/supported" target="_blank" rel="noopener noreferrer" class="">x402 Facilitator Supported Networks</a></strong> — Lists supported networks (including <code>stellar:testnet</code>) and facilitator configuration.</li>
</ul>
<h3 class="anchor anchorTargetStickyNavbar_Vzrq" id="build-on-stellar-relayer-with-openzeppelin-x402-plugin">Build on Stellar Relayer (with OpenZeppelin x402 Plugin)<a href="#build-on-stellar-relayer-with-openzeppelin-x402-plugin" class="hash-link" aria-label="Direct link to Build on Stellar Relayer (with OpenZeppelin x402 Plugin)" title="Direct link to Build on Stellar Relayer (with OpenZeppelin x402 Plugin)" translate="no">​</a></h3>
<p>The <a href="https://github.com/OpenZeppelin/relayer-plugin-x402-facilitator" target="_blank" rel="noopener noreferrer" class="">Relayer Plugin for x402</a> implements the x402 Facilitator API so you can serve x402 payments directly from a Relayer instance. The x402 Facilitator leverages the OpenZeppelin Relayer Framework. It works with the <a href="https://github.com/coinbase/x402" target="_blank" rel="noopener noreferrer" class="">Coinbase x402</a> ecosystem and exposes the expected <code>/verify</code>, <code>/settle</code>, and <code>/supported</code> endpoints under the Relayer plugin router.</p>
<h4 class="anchor anchorTargetStickyNavbar_Vzrq" id="how-to-use">How to Use<a href="#how-to-use" class="hash-link" aria-label="Direct link to How to Use" title="Direct link to How to Use" translate="no">​</a></h4>
<p>This x402 facilitator service is available on Testnet and Mainnet. Under the hood, the plugin leverages OpenZeppelin Channels to submit transactions onchain via a managed Relayer and Facilitator setup.</p>
<h4 class="anchor anchorTargetStickyNavbar_Vzrq" id="testnet">Testnet<a href="#testnet" class="hash-link" aria-label="Direct link to Testnet" title="Direct link to Testnet" translate="no">​</a></h4>
<p>To use the facilitator on testnet, you will need:</p>
<ul>
<li class="">An API Key (Relayer Service). Generate your testnet API key here: <a href="https://channels.openzeppelin.com/testnet/gen" target="_blank" rel="noopener noreferrer" class="">https://channels.openzeppelin.com/testnet/gen</a></li>
<li class="">Facilitator URL. Use the following facilitator endpoint in your configuration: <code>https://channels.openzeppelin.com/x402/testnet</code></li>
</ul>
<h4 class="anchor anchorTargetStickyNavbar_Vzrq" id="mainnet">Mainnet<a href="#mainnet" class="hash-link" aria-label="Direct link to Mainnet" title="Direct link to Mainnet" translate="no">​</a></h4>
<p>To use the Facilitator on mainnet, you will need:</p>
<ul>
<li class="">An API Key (Relayer Service). Generate your mainnet API key here: <a href="https://channels.openzeppelin.com/gen" target="_blank" rel="noopener noreferrer" class="">https://channels.openzeppelin.com/gen</a></li>
<li class="">Facilitator URL. Use the following facilitator endpoint in your configuration: <code>https://channels.openzeppelin.com/x402</code></li>
</ul>
